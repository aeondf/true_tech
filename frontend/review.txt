ЗАПУСК И РАЗМЕЩЕНИЕ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Локальный запуск (текущий):
  python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
  (запускать из директории backend/)

Base URL:        http://localhost:8000/v1
Swagger UI:      http://localhost:8000/docs
ReDoc:           http://localhost:8000/redoc

Docker Compose (полный стек):
  docker compose up -d
  Сервисы: backend:8000, postgres:5432, redis:6379,
           ollama:11434, media-service:8010, frontend:3001, openwebui:3000

Зависимости (должны быть запущены):
  - PostgreSQL 16 + pgvector (порт 5432)
  - Redis 7 (порт 6379)
  - Доступ к MWS API (https://api.gpt.mws.ru)

БАЗА ДАННЫХ — СХЕМА
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Расширение: pgvector (embedding размерность = 1024)
Все таблицы создаются автоматически при старте приложения.

ТАБЛИЦЫ:
  users
    id          String (PK)
    created_at  DateTime

  conversations
    id          String (PK)
    user_id     String (FK → users.id)
    title       String(512), nullable
    created_at  DateTime
    updated_at  DateTime

  messages
    id                String (PK)
    conversation_id   String (FK → conversations.id)
    role              String(16) — "user" | "assistant" | "system"
    content           Text
    token_count       Integer, nullable
    created_at        DateTime

  memories
    id                String (PK)
    user_id           String (FK, indexed)
    conversation_id   String, nullable
    text              Text
    embedding         Vector(1024)  — IVFFlat индекс (cosine)
    created_at        DateTime

  file_chunks
    id            String (PK)
    file_id       String (indexed)
    user_id       String (indexed)
    filename      String(512)
    chunk_index   Integer
    text          Text
    embedding     Vector(1024)  — IVFFlat индекс (cosine)

  router_logs
    id               String (PK)
    user_id          String (indexed)
    message_preview  String(512)
    task_type        String(64)
    model_id         String(128)
    confidence       Float
    latency_ms       Integer, nullable
    created_at       DateTime

  Response 200:
  {
    "status": "ok" | "degraded",
    "services": {
      "postgres": true,
      "mws_api": true,
      "image_gen": "...",
      "asr": "..."
    }
  }

──────────────────────────────────────────
2. ЧАТ (OpenAI-compatible)
──────────────────────────────────────────

POST /v1/chat/completions
  Основной эндпоинт. Поддерживает streaming.
  Если model="auto" — роутер сам выбирает модель.

  Request:
  {
    "model": "auto" | "mws-gpt-alpha" | "qwen3-coder-480b-a35b" | ...,
    "messages": [
      {"role": "system", "content": "..."},
      {"role": "user", "content": "..."}
    ],
    "temperature": 0.7,        // опционально
    "max_tokens": 2000,        // опционально
    "stream": false,           // true = SSE
    "user": "user_id",         // опционально, для памяти и истории
    "conversation_id": "...",  // опционально
    "attachments": [           // опционально, для файлов/изображений
      {
        "name": "doc.pdf",
        "mime": "application/pdf",
        "data": "<base64>"     // опционально
      }
    ]
  }

  Response (stream=false):
  {
    "choices": [{"message": {"role": "assistant", "content": "..."}, "finish_reason": "stop"}],
    "usage": {"prompt_tokens": 10, "completion_tokens": 50, "total_tokens": 60}
  }

  Response (stream=true) — Server-Sent Events:
    data: {"choices":[{"delta":{"content":"Hello"}}]}
    data: {"choices":[{"delta":{"content":" there"}}]}
    data: [DONE]

  HTTP 429 — лимит запросов (60/мин по IP)
  HTTP 503 — MWS API недоступен

POST /v1/completions
  Raw completion (без формата чата).
  Request: {"model": "...", "prompt": "...", "max_tokens": 500, "stream": false}
  Response: стандартный OpenAI completion format

POST /v1/embeddings
  Генерация эмбеддингов через bge-m3.
  Request:  {"model": "bge-m3", "input": "текст" | ["текст1", "текст2"]}
  Response: {"data": [{"object": "embedding", "embedding": [...], "index": 0}], ...}

GET /v1/models
  Список доступных моделей от MWS.
  Response: {"data": [{"id": "mws-gpt-alpha", "object": "model", "owned_by": "mws"}, ...]}

──────────────────────────────────────────
3. ФАЙЛЫ / RAG
──────────────────────────────────────────

POST /v1/files/upload
  Загрузка документа для RAG (PDF, DOCX, TXT).
  Request: multipart/form-data
    file:    <бинарный файл>
    user_id: "user123"   (опционально, default: "anonymous")
  Response:
  {
    "file_id": "uuid",
    "chunks": 15
  }
  Ограничения: до 50 МБ, chunk_size=2048 символов

──────────────────────────────────────────
4. ИСТОРИЯ ДИАЛОГОВ
──────────────────────────────────────────

GET /v1/history/{user_id}
  Список всех диалогов пользователя (новые первые).
  Query: limit=50
  Response:
  {
    "conversations": [
      {"id": "...", "title": "...", "created_at": "...", "updated_at": "..."}
    ]
  }

GET /v1/history/{user_id}/{conversation_id}
  Все сообщения в диалоге.
  Query: limit=200
  Response:
  {
    "conversation_id": "...",
    "messages": [
      {"id": "...", "role": "user|assistant|system", "content": "...",
       "token_count": 150, "created_at": "..."}
    ]
  }

PATCH /v1/history/{user_id}/{conversation_id}
  Переименовать диалог.
  Request:  {"title": "Новое название"}
  Response: {"id": "...", "title": "Новое название"}

DELETE /v1/history/{user_id}/{conversation_id}
  Удалить диалог со всеми сообщениями.
  Response: {"deleted": true}

5. ПАМЯТЬ (MEMORY)
──────────────────────────────────────────

POST /v1/memory/{user_id}/search
  Семантический поиск по памяти пользователя.
  Request:  {"query": "что любит пользователь?", "top_k": 10}
  Response:
  {
    "user_id": "...",
    "results": [{"id": "...", "text": "...", "score": 0.87}]

GET /v1/memory/{user_id}
  Все записи памяти пользователя.
  Query: limit=50
  Response:
  {
    "user_id": "...",
    "memories": [
      {"id": "...", "text": "...", "conversation_id": "...|null", "created_at": "..."}
    ]
  }

DELETE /v1/memory/{user_id}/{memory_id}
  Удалить одну запись памяти.
  Response: {"deleted": "mem_uuid"}

DELETE /v1/memory/{user_id}
  Удалить всю память пользователя.
  Response: {"deleted_count": 42}

──────────────────────────────────────────
6. ГОЛОС
──────────────────────────────────────────

POST /v1/voice/message
  Аудио → ASR → LLM → TTS → аудио-ответ.
  Request: multipart/form-data
    audio:   <WAV/MP3/OGG/FLAC/M4A>
    user_id: "user123"  (опционально)
  Response (успех):
    Content-Type: audio/wav или audio/mpeg
    Headers:
      X-Transcript: <urlencode текст запроса>
      X-Answer:     <urlencode текст ответа LLM>
    Body: <бинарное аудио>
  Response (TTS недоступен):
    {"transcript": "...", "answer": "...", "tts_available": false}
  Пайплайн: Whisper (MWS) → LLM (max 300 токенов) → Silero/edge-tts

WebSocket /v1/ws/voice
  Стриминговый голосовой интерфейс.
  Соединение: ws://localhost:8000/v1/voice/ws
  Протокол:
    Клиент → Сервер:  бинарный аудио-блоб
    Сервер → Клиент:  {"type": "transcript", "text": "..."}
    Сервер → Клиент:  {"type": "token", "text": "..."} (повторяется)
    Сервер → Клиент:  бинарный MP3
    (fallback):       {"type": "done", "text": "...", "tts_available": false}

──────────────────────────────────────────
7. ИЗОБРАЖЕНИЯ / VLM
──────────────────────────────────────────

POST /v1/image/generate
  Генерация изображения по промпту.
  Request:
  {
    "prompt": "Закат над океаном",
    "model": "qwen-image",          // опционально
    "size": "1024x1024"             // опционально: 512x512, 1024x1024, 2048x2048
  }
  Response:
  {
    "data": [{"url": "https://...", "revised_prompt": "..."}]
  }
  Fallback (если недоступен):
  {"error": "...", "fallback": true, "description": "Генерация недоступна. Промпт: ..."}

POST /v1/vlm/analyze
  Анализ изображения через vision-модель.
  Request:
  {
    "image_url": "https://example.com/img.jpg",
    "question": "Что изображено на фото?",
    "model": "qwen2.5-vl"           // опционально
  }
  Response:
  {
    "choices": [{"message": {"content": "На изображении..."}}]
  }
  Fallback: {"answer": null, "fallback": true, "message": "..."}

──────────────────────────────────────────
9. ГЛУБОКОЕ ИССЛЕДОВАНИЕ (Deep Research)
──────────────────────────────────────────

POST /v1/research
  Многоэтапный поиск с SSE прогрессом.
  Request:  {"query": "Последние разработки в ИИ?", "user_id": "user123"}
  Response: Server-Sent Events

    event: progress
    data: {"step": 1, "message": "Генерирую подзапросы…"}

    event: progress
    data: {"step": 2, "sub_queries": ["...", "...", "..."]}

    event: progress
    data: {"step": 3, "message": "Ищу и парсю страницы…"}

    event: progress
    data: {"step": 4, "pages_fetched": 12}

    event: progress
    data: {"step": 5, "message": "Синтезирую ответ…"}

    event: done
    data: {"answer": "Развёрнутый ответ со ссылками..."}

  Таймауты: подзапросы 30с, поиск+парсинг 15с, всего 60с, синтез 60с
  Ограничения: до 10 страниц, до 2000 символов с каждой

РОУТЕР ЗАДАЧ (автоматический выбор модели)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

При model="auto" роутер определяет тип задачи:

  text           → mws-gpt-alpha        — обычный текст
  code           → qwen3-coder-480b-a35b — код (regex: def/class/функци...)
  web_search     → mws-gpt-alpha + web  — URL в сообщении
  web_parse      → mws-gpt-alpha + web  — URL + парсинг
  deep_research  → qwen2.5-72b-instruct — ключевые слова: "исследуй/подробно/анализ..."
  file_qa        → mws-gpt-alpha + RAG  — вложенный PDF/DOCX/TXT
  asr            → whisper-turbo-local  — аудио вложение
  vlm            → qwen2.5-vl           — изображение во вложении
  image_gen      → qwen-image           — "нарисуй" / "generate image"

Приоритет: аудио > изображение > файл > research > URL > код > LLM fallback

ФОРМАТ ОШИБОК
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{
  "error": {
    "code": 503,
    "message": "MWS API недоступен",
    "detail": "..."
  }
}

Коды:
  200 — успех
  400 — плохой запрос (валидация)
  404 — не найдено (диалог, память, etc.)
  413 — файл слишком большой (> 50 МБ)
  415 — неподдерживаемый тип файла
  429 — превышен rate limit
  500 — внутренняя ошибка
  503 — MWS API недоступен

ДОПОЛНИТЕЛЬНО
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- Память автоматически инжектируется в системный промпт при чате
- Контекст сжимается если > 8000 токенов (суммаризация старых сообщений)
- Эмбеддинги кэшируются в Redis (TTL 1ч, ключ mem:{user_id}:{hash(query)})
- RAG: косинусное сходство через pgvector, IVFFlat индекс
- Web-инструменты (search + parse) выполняются параллельно при deep_research
- Алembic миграции: alembic upgrade head
