# Task Router Specification

## Executive Summary
Трёхпроходный детерминированный + LLM роутер для MWS AI Gateway. Классифицирует входящий запрос по task_type и выбирает модель. Работает как независимый сервис внутри proxy.py.

## Functional Requirements

### Выходной формат
```json
{
  "task_type": "code",
  "model_id": "qwen3-coder-480b-a35b",
  "confidence": 1.0,
  "which_pass": 1
}
```

### Task Types и модели
| task_type     | model_id                  |
|---------------|---------------------------|
| text          | qwen2.5-72b-instruct      |
| code          | qwen3-coder-480b-a35b     |
| asr           | whisper-turbo-local        |
| vlm           | cotype-pro-vl-32b         |
| image_gen     | qwen-image                |
| web_search    | qwen2.5-72b-instruct      |
| web_parse     | qwen2.5-72b-instruct      |
| deep_research | qwen2.5-72b-instruct      |
| file_qa       | qwen2.5-72b-instruct      |

### Проход 1 — MIME/расширение (O(1), confidence=1.0)
- `audio/*` или `.mp3/.wav/.ogg/.m4a/.flac` → `asr`
- `image/*` или `.png/.jpg/.jpeg/.webp/.gif` → `vlm`
- `.pdf/.docx/.txt` → `file_qa`

### Проход 2 — структурный анализ (O(n), confidence=0.9)
- RESEARCH_KEYWORDS (исследу, глубокий анализ, изучи...) → `deep_research`
- IMAGE_GEN_KEYWORDS (нарисуй, сгенерируй изображение...) → `image_gen`
- URL в тексте + явный запрос парсить (открой, прочитай, проанализируй ссылку) → `web_parse`
- URL в тексте без явного запроса + поисковый intent → `web_search`
- CODE_KEYWORDS (код, функци, class, def, SELECT...) → `code`
- Всё остальное → `text`

### Проход 3 — LLM классификатор (fallback, confidence из LLM)
- Модель: `llama-3.1-8b-instruct` через MWS API
- Таймаут: 60 секунд
- При ошибке/таймауте → `text`, confidence=0.5
- Если LLM вернул confidence < 0.7 → переводим в `text`

### deep_research поведение
Роутер возвращает `task_type: "deep_research"` клиенту. Клиент сам делает запрос на `/v1/research`. proxy.py не перенаправляет автоматически.

## Логирование
Postgres, только таблица `router_log`:
```sql
CREATE TABLE router_log (
    id UUID PRIMARY KEY,
    user_id TEXT,
    message_preview TEXT,  -- первые 512 символов
    task_type TEXT,
    model_id TEXT,
    confidence FLOAT,
    which_pass INTEGER,    -- 1, 2 или 3
    latency_ms INTEGER,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## RAG (pgvector)
- Хранилище: PostgreSQL + pgvector
- Используется при `file_qa`
- Эмбеддинги: `bge-m3` через MWS

## Out of Scope
- Авто-redirect с /chat/completions на /research
- Rate limiting
- Кэширование решений роутера
