# True Tech Hack — AI Gateway Backend

Умный ассистент с роутингом, памятью, RAG, голосом и генерацией медиа.

## Быстрый старт

```bash
cp .env.example .env
# Вставь MWS_API_KEY в .env
docker compose up -d
```

Через ~30 секунд доступно:
- **API**: http://localhost:8000/docs
- **OpenWebUI**: http://localhost:3000

## Что умеет

| Запрос | Модель |
|--------|--------|
| Обычный текст | mws-gpt-alpha |
| Код | kodify-2.0 |
| Исследуй / глубокий анализ | cotype-preview-32k |
| Голосовое сообщение | faster-whisper → LLM → edge-tts |
| Анализ изображения | LLaVA |
| Генерация изображения | Stable Diffusion |

## Ключевые эндпоинты

```
POST /v1/chat/completions     — основной чат (OpenAI-совместимый)
POST /v1/files/upload         — загрузить PDF/DOCX/TXT для RAG
POST /v1/voice/message        — голосовое сообщение → MP3-ответ
POST /v1/research             — Deep Research с SSE-прогрессом
POST /v1/tools/generate-pptx  — сгенерировать PowerPoint
POST /v1/tools/web-search     — поиск в интернете
GET  /v1/health               — статус всех сервисов
```

## Разработка

```bash
make run       # локальный запуск с hot-reload
make migrate   # применить миграции БД
make test      # запустить тесты
make lint      # проверка кода
```

## Статус реализации

### Сделано

- **Роутер** — определяет тип запроса (текст / код / исследование / голос / картинка) детерминированными правилами, при неуверенности спрашивает локальную модель Qwen. Каждое решение логируется в БД
- **Чат-прокси** — OpenAI-совместимый эндпоинт, работает с OpenWebUI из коробки
- **Память** — извлекает факты о пользователе из разговоров, хранит векторно, подкладывает в контекст автоматически
- **Сжатие контекста** — если история чата стала слишком длинной (> 8000 токенов), LLM делает краткое изложение старых сообщений
- **RAG** — загрузи PDF/DOCX/TXT, задавай вопросы по документу
- **Голос** — принимает аудио, возвращает MP3-ответ; есть и WebSocket-режим для стриминга
- **Deep Research** — по одному запросу генерирует 5 поисковых подзапросов, параллельно ищет и читает страницы, синтезирует итоговый ответ с источниками; прогресс стримится через SSE
- **Генерация презентаций** — `POST /v1/tools/generate-pptx` создаёт .pptx файл по теме
- **Поиск и парсинг** — DuckDuckGo поиск + чтение любой веб-страницы
- **Картинки** — генерация через Stable Diffusion, анализ через LLaVA; при недоступности сервиса — понятный fallback вместо ошибки
- **Health-check** — `/v1/health` показывает статус каждого сервиса
- **База данных** — PostgreSQL + pgvector, IVFFlat индексы для быстрого векторного поиска
- **Кэш** — Redis для ускорения поиска по памяти
- **Rate limiter** — 60 запросов в минуту на IP
- **Docker** — один `docker compose up -d` поднимает всё
